# سجل قرارات Phase 2

## قرارات البيئة

- تم اعتماد Python 3.11.9، 64-bit، AMD64 على Windows كما أبلغ Python نفسه.
- كانت `.venv` موجودة قبل المرحلة، وأثبت `pyvenv.cfg` والفحص التنفيذي أنها بيئة سليمة تخص جذر المشروع؛ لذلك أُعيد استخدامها ولم تُنشأ فوقها بيئة جديدة.
- استُخدم `.venv\Scripts\python.exe` مباشرةً في كل عمليات pip والتحقق، ولم يُستخدم pip العام أو `pip --user`.
- لا يعتمد المشروع على الأمر `py` لأن Microsoft Python Launcher غير متاح، وهذا لا يؤثر في سير العمل.
- يتعذر على صندوق Codex المعزول تشغيل Microsoft Store shim بسبب خطأ جلسة Windows، بينما نجح المفسر نفسه خارج العزل. سُجل هذا كاختلاف وصول للأداة، لا كغياب Python عند المستخدم.

## قرارات الاعتماديات

- روجعت الحزم الخمس عشرة ونطاقاتها، وكل حزمة مرتبطة باستخدام مخطط:

| Dependency | الاستخدام المخطط |
|---|---|
| `numpy` | المصفوفات والعمليات العددية وتمثيلات السلة |
| `pandas` | قراءة البيانات وتنظيفها وتجميع المعاملات |
| `scipy` | المصفوفات المتناثرة والحسابات العلمية |
| `scikit-learn` | أدوات التقييم وإعادة أخذ العينات ومساندة التجارب |
| `mlxtend` | FP-Growth وApriori وتوليد قواعد الارتباط |
| `matplotlib` | الرسوم الأكاديمية الثابتة |
| `plotly` | التصورات التفاعلية |
| `networkx` | شبكة ارتباط المنتجات |
| `ipywidgets` | عناصر التحكم في المختبر التفاعلي |
| `jupyterlab` | بيئة الدفاتر الرئيسة |
| `notebook` | تشغيل وتوافق واجهة Notebook |
| `nbformat` | فحص بنية الدفاتر ومعالجتها |
| `openpyxl` | قراءة وكتابة Excel عند الحاجة |
| `psutil` | قياس الذاكرة والعملية تقريبياً |
| `tqdm` | إظهار تقدم العمليات الطويلة |

- لم يتطلب `requirements.txt` أي تعديل؛ حل pip نطاقاته على Python 3.11 بلا تعارض.
- رُقيت `pip` و`setuptools` و`wheel` داخل `.venv` فقط.
- أُنشئ `requirements-resolved.txt` من `pip freeze --all` لتسجيل البيئة العاملة بدقة.
- نجح `pip check` وكل استيراد مباشر، بما في ذلك واجهات FP-Growth وApriori وAssociation Rules.

## قرارات بحث البيانات

- اقتصر البحث على صفحات UCI الرسمية، ووثائق CRAN الرسمية، وصفحات dunnhumby الأصلية، وصفحة مسابقة Instacart الأصلية على Kaggle وشروطها.
- رُشحت **UCI Online Retail** كمجموعة أساسية بسبب وضوح الترخيص وبنية الفاتورة وأسماء المنتجات والكمية والزمن والسعر والبلد ومؤشر الإلغاء.
- رُشحت **Groceries من حزمة arules** كبديل صغير ومستقل عن UCI، بشرط حسم حقوق إعادة توزيع البيانات قبل تنزيلها أو إدراجها.
- لم تُعتبر `Online Retail II` بديلاً عملياً عند عجز الجهاز عن معالجة Online Retail، لأنها أكبر ومن المصدر نفسه؛ لكنها مرشح غني لتحليل زمني أطول.
- اعتُبرت `The Complete Journey` وInstacart غنيتين للميزات الإبداعية، لكن حجمهما وتعقيدهما وقيود الاستخدام أو الوصول تجعلهما أقل ملاءمة لنسخة الطالب الأساسية.
- لم يُنزّل أي ملف بيانات، ولم تُحسب أي إحصاءات من البيانات نفسها.

## الخيارات المرفوضة أو المؤجلة

- **مرايا Kaggle غير الأصلية:** رُفضت عندما توفر المصدر الأصلي، لأن المرآة قد تغيّر الملف أو الترخيص أو المعالجة.
- **FIMI Retail benchmark:** مؤجل كمرشح أداء فقط؛ العناصر أرقام مجهولة بلا أسماء، والترخيص التفصيلي غير ظاهر بوضوح، فلا يدعم التفسير التجاري أو المحاكي جيداً.
- **بيانات اصطناعية صغيرة:** رُفضت كمصدر أساسي لأنها لا تحقق هدف المشروع في تحليل معاملات حقيقية.
- **Online Shoppers Purchasing Intention:** رُفضت لأنها بيانات جلسات وصفحات وليست سلال منتجات يمكن استخراج قواعد شراء منها.
