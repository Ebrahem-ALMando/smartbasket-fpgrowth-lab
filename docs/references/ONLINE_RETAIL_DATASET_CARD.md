## النطاق والمخطط المرصودان فعلياً

يحتوي workbook على ورقة واحدة باسم `Online Retail`، و541,909 سطراً و8 أعمدة:

| العمود | النوع الدلالي المستخدم | الوصف المختصر |
|---|---|---|
| InvoiceNo | string | معرف الفاتورة؛ البادئة `C` تشير إلى الإلغاء |
| StockCode | string | معرف المنتج أو السطر الإداري |
| Description | string | وصف المنتج كما ورد في المصدر |
| Quantity | integer | كمية السطر، وقد تكون سالبة في الخام |
| InvoiceDate | datetime | وقت الفاتورة |
| UnitPrice | float | سعر الوحدة بالجنيه الإسترليني |
| CustomerID | string | معرف العميل، ويسمح بغيابه |
| Country | string | بلد المعاملة |

التغطية الزمنية الخام من `2010-12-01 08:26` إلى `2011-12-09 12:50`. رُصدت 25,900 قيمة فاتورة و4,070 قيمة `StockCode` و38 بلداً في البيانات الخام.

## جودة البيانات المرصودة

القيم المفقودة هي: `Description` عدد 1,454 و`CustomerID` عدد 135,080، وبقية الأعمدة صفر. تتعارض هذه النتيجة مع وصف UCI الذي يذكر عدم وجود قيم مفقودة؛ تعتمد البطاقة ومرحلة التجهيز على workbook الفعلي.

كما رُصدت:

- 5,268 نسخة صف كامل زائدة، و10,147 صفاً مشاركاً في مجموعات تكرار.
- 9,288 سطر إلغاء موزعة على 3,836 فاتورة ملغاة.
- 10,624 سطراً بكمية سالبة، منها 1,336 لا تندرج أولاً تحت الإلغاء؛ لا توجد كمية صفرية.
- سعران سالبان و2,515 سعراً صفرياً، أي 2,517 سعراً غير صالح إجمالاً.
- أسطر بريد وخصم ورسوم وتسويات يدوية وأكواد قسائم، مصنفة فقط من القيم المرصودة.

## المعالجة والمخرجات

بعد تطبيق ترتيب سياسة الفصل وإزالة النسخ الكاملة الزائدة بقي 522,447 سطر شراء صالح وفُصل 19,462 سطراً. أعيد بناء 19,773 معاملة صالحة تحتوي 3,797 منتجاً عبر جميع البلدان. تتصالح الأعداد: `522,447 + 19,462 = 541,909`.

النطاق الأساسي المختار هو **United Kingdom**: 17,901 معاملة و3,791 منتجاً. حُفظ تمثيل السلال كمصفوفة SciPy CSR ثنائية بأبعاد `17,901 × 3,791` و473,636 قيمة غير صفرية. تحتفظ المخرجات المنفصلة بجميع البلدان لأغراض التدقيق وتحليل الحساسية.

للتفاصيل الدقيقة راجع `DATA_CLEANING_POLICY.md` و`PHASE_3_DECISION_LOG.md` وجداول lineage؛ لا تعيد هذه البطاقة سرد السياسة كاملة.

## القيود والاستخدام المسؤول

- البيانات تخص بائع تجزئة غير متجري في المملكة المتحدة، ولا تمثل جميع قطاعات التجزئة أو البلدان.
- الفواتير لا تثبت السببية، والارتباط المشترك لا يعني أن منتجاً يسبب شراء الآخر.
- غياب `CustomerID` واسع، لذلك لا تصلح البيانات المعالجة هنا لاستنتاجات سلوكية على مستوى العميل من دون سياسة إضافية.
- التصنيف الإداري محافظ وقد يحتاج مراجعة عند تغيير المصدر أو الغرض.
- اختيار المملكة المتحدة يحسن اتساق السوق لكنه يحد من تعميم النتائج دولياً.
- لم تُنفذ في Phase 3 خوارزميات FP-Growth أو Apriori، ولم تُولد قواعد ارتباط أو نتائج WEKA.
